P0 — Bloqueia adoção, segurança, compliance ou gera erro operacional grave
 * Exposição de caminhos locais em Logs: O arquivo audit_20260326.txt revela caminhos absolutos do host (C:\Users\<username>\...) em logs de produção/amostra. → Correção: Implementar sanitização de paths no logger ou usar caminhos relativos ao volume de dados para evitar Information Leakage da estrutura do host do auditor.
 * Inconsistência de versão de Runtime: O Dockerfile utiliza python:3.13-slim, mas o sonar-project.bin e o pyproject.bin referenciam 3.12. → Correção: Unificar a versão base para 3.13 em todos os artefatos de CI/CD para evitar comportamentos inesperados de bibliotecas que dependem de C-extensions (como cryptography ou libpq-dev).
 * Risco de Falso Negativo em Regéx: No varios-settings.bin, o padrão para CEP_BR (\b\d{5}-?\d{3}\b) é muito genérico e pode capturar IDs de produtos ou sequências numéricas aleatórias. → Correção: Adicionar validação de contexto (ex: proximidade com termos como "CEP" ou "Endereço") na lógica do scanner para reduzir o ruído em auditorias de larga escala.
P1 — Friction alta: onboarding, deploy, troubleshooting, ou contradiz outro doc
 * Dualidade de nomes (Data Boar vs lgpd_crawler): O pyproject.bin define o nome do projeto como python3-lgpd-crawler, mas a documentação foca em Data Boar. Isso causa confusão no pip install. → Correção: Padronizar o nome de distribuição no pyproject.toml ou deixar explícito no README que o comando data_boar é um entry-point do pacote lgpd-crawler.
 * Dependências de Sistema não documentadas: O Dockerfile instala diversas bibliotecas de desenvolvimento (libpq-dev, unixodbc-dev, etc). Um usuário tentando rodar o projeto localmente via uv sync ou pip sem essas libs de sistema terá erros de compilação. → Correção: Criar uma seção "Prerequisites" no README listando as dependências de sistema para Debian/Ubuntu e RHEL.
 * Persistência de dados: O Dockerfile define ENV CONFIG_PATH=/data/config.yaml, mas não há um VOLUME explícito ou instrução clara sobre como persistir o banco de dados SQLite/Auditoria entre restarts do container. → Correção: Adicionar instrução de docker run -v no guia de uso.
