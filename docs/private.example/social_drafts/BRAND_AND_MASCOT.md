# Social posts — brand and mascot (template)

**Purpose:** When drafting LinkedIn / X / blog posts about **Data Boar**, keep **name + mascot + narrative** aligned with the public [README.md](../../README.md) (data soup, sniffing/finding ingredients, optional OCR).

**Tracked / public-safe:** no operator-only paths; copy ideas into your **gitignored** `docs/private/` drafts.

## Every post (light touch)

1. **Name:** **Data Boar** early in the hook or first paragraph.
2. **Mascot:** One **🐗** per post is enough for text-first networks (LinkedIn). Avoid stuffing emoji.
3. **Optional image:** Repository asset `api/static/mascot/data_boar_mascote_color.svg` — export to PNG if the network does not accept SVG for uploads.
4. **Vocabulary (pick one phrase if it fits):** *data soup*, *ingredients*, *sniff / faro* — same story as the README; do not repeat every sentence.
5. **Version numbers:** If you mention **“current release”**, **“latest on Docker Hub”**, or a **semver**, it must match the **[README.md](../../README.md)** **Current release** line and **[docs/ops/today-mode/PUBLISHED_SYNC.md](../../ops/today-mode/PUBLISHED_SYNC.md)** (what customers can actually install). For **upcoming** work, say so explicitly.

## Revision markers

In draft `.md` files, lines ending with **`<-`** are **operator notes** (what to fix before paste). Remove them in the published text.
