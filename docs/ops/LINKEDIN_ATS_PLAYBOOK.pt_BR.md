Este texto permanece **genérico**: sem nomes de empregadores, salários nem texto copiável atado a uma pessoa.

## 1. Objetivos (o que é “bom”)

1. **Clareza:** Recrutadores e gestores entendem **papel**, **senioridade** e **prova** em menos de 30 segundos.
2. **Match com ATS:** CV exportado e LinkedIn compartilham **palavras-chave coerentes** com a trilha alvo (sem empilhar termos vazios).
3. **Confiança:** Afirmações batem com **evidência** (repositórios, releases, documentação, resultados mensuráveis). Evitar overclaim de compliance ou jurídico.
4. **Sustentabilidade:** Um **hábito semanal pequeno** vence uma reescrita única.

---

## 2. Social Selling Index (SSI) no LinkedIn

O **SSI** do LinkedIn é uma **métrica de painel** (não é uma nota pública no perfil). Agrupa **quatro pilares** (rótulos podem variar um pouco ao longo do tempo):

1. Construir **marca profissional**
2. Encontrar as **pessoas certas**
3. Interagir com **insights**
4. Construir **relacionamentos**

**Uso ético:** Melhore **visibilidade e relacionamentos de verdade**. Não faça spam de convites, engajamento falso nem skills inventadas.

**Hábitos práticos (cerca de 30 minutos por semana):**

- Atualize **uma** seção por semana (headline → Sobre → experiência principal → destaques).
- Publique ou comente **uma** vez com substância (insight técnico, release, lição aprendida) a cada 1–2 semanas.
- Trate o SSI como **guia**, não como garantia de contratação.

---

## 3. Mecânica de ATS (arquivos de currículo)

- Prefira **uma coluna**, **títulos de seção padrão** (Experiência, Formação, Habilidades) e **marcadores simples**.
- Exporte **PDF** a partir da mesma fonte de verdade do LinkedIn (evite Word divergente do PDF).
- Padrão de nome de arquivo: **`Cargo_Sobrenome_Nome.pdf`** ou convenção local; evite caracteres especiais.
- Espelhe **5–10 palavras-chave** centrais da vaga-alvo de forma **natural** nas experiências (não um bloco gigante de keywords).

## 4. LinkedIn seção a seção

### 4.1 Foto e capa

- Foto: profissional, legível em miniatura.
- Capa: opcional; pode reforçar **domínio** (privacidade, segurança, plataforma de dados) sem poluir.

### 4.2 Headline

- Fórmula: **`Papel | Resultado | Domínio`** (forma de exemplo: *Engenheiro de backend | Ferramentas de proteção de dados | Python / CI*).
- Evite clichês vazios (“apaixonado”, “guru”). Prefira **verbos e domínios** que recrutadores buscam.

### 4.3 Sobre

- Duas primeiras linhas: **quem você é** e **o que entrega**.
- Meio: **prova** (produtos, métricas, stack, consciência de governança).
- Fechamento: **o que busca em seguida** (tipo de papel, remoto/híbrido, escopo) sem soar desesperado.

### 4.4 Experiência

- Bullets estilo **STAR** quando der: situação, tarefa, ação, **resultado mensurável**.
- Conecte o trabalho a **segurança, qualidade e manutenção** quando for verdade (testes, CI, revisões).

### 4.5 Habilidades e recomendações de habilidade

- Fixe **3–5** habilidades alinhadas ao **papel-alvo**.
- Ordene para que o topo combine **ATS + realidade** (linguagens, frameworks, nuvem, segurança).

### 4.6 Licenças e certificações

- Liste credenciais **verificáveis**; evite selos expirados salvo ainda relevantes e **marcados** como tal.

### 4.7 Destaques

- Linke **artefatos públicos**: releases no GitHub, palestras, **documentação** que você escreveu, casos sem dados sensíveis de cliente.

### 4.8 Recomendações

- **Escreva** recomendações bem fundamentadas para gerar reciprocidade ao longo do tempo; **peça** a pares que conhecem seu trabalho.

## 5. Pacotes de palavras-chave por arquétipo (A–F)

Use como **sementes**; personalize com stack e projetos reais.

| Arquétipo | Exemplos de sementes de palavras-chave |
| --------- | -------------------------------------- |
| **A** Backend / Python | Python, pytest, design de API, conectores, desempenho, typing, empacotamento |
| **B** API / web | REST, OpenAPI, authn/z, segurança HTTP, front básico, UX para admins |
| **C** DevOps / CI | GitHub Actions, Docker, cadeia de suprimentos, SBOM, release engineering, SRE |
| **D** Documentação / narrativa de compliance | Redação técnica, linguagem consciente de LGPD, evidência, linguagem de risco, bilíngue EN/pt-BR |
| **E** AppSec | Modelagem de ameaças, CodeQL, risco de dependências, padrões seguros, divulgação responsável |
| **F** QA / testes | pytest, fixtures, estratégia de regressão, gates de CI, redução de flakiness |

## 6. Cadência

| Cadência | Ações |
| -------- | ----- |
| **Semanal (~30 min)** | Um ajuste no perfil; um insight ou comentário; revisar uma descrição de vaga para alinhar palavras-chave. |
| **Mensal** | Reler Sobre e headline; atualizar Destaques; exportar PDF novo do CV. |
| **Trimestral** | Auditoria de habilidades vs papel-alvo; trilha de certificação só se **uma** frente estiver ativa (veja documento de roteiros de aprendizado). |
