# Pre-release 1.7.1-beta

**Kind:** **Pre-release** (`-beta` per [VERSIONING.md](../VERSIONING.md)) — **not** the same baseline as **1.7.0** on Docker Hub / last stable GitHub Release.

**In-repo date:** 2026-04-08.

**Version string:** **`1.7.1-beta`** — **build** bump from **1.7.0** with **beta** suffix until the slice is promoted to **`-rc`** or **1.7.1** final.

---

## Why this exists

Clones of `main` after the locale/dashboard work should report a version that **does not read as 1.7.0**. Operators comparing **`python main.py --version` / About** to **`fabioleitao/data_boar:1.7.0`** must see that **`main` has moved**.

---

## Highlights (vs 1.7.0)

- **Dashboard i18n (M-LOCALE-V1):** HTML under **`/{locale_slug}/…`** (e.g. **`/en/`**, **`/pt-br/`**); JSON message catalogs **`en`** + **`pt-BR`**; negotiation (**cookie** → **`Accept-Language`** → **`locale.default_locale`**); unprefixed **`/`**, **`/config`**, **`/reports`**, **`/help`**, **`/about`** redirect. JSON API routes stay **without** a locale prefix.
- **`enterprise_surface`:** `GET /status` and `GET /health` expose transport + license trust + global API-key posture (**RBAC** remains **not** per-route).
- **Docs / PMO:** USAGE, TECH_GUIDE, PLAN_DASHBOARD_I18N / access-control matrix; optional **`locale`** block in **`deploy/config.example.yaml`**.

## Docker (optional pre-release tag)

When you publish a **beta** image (does **not** replace **`latest`** unless you explicitly choose to):

```powershell
.\scripts\docker-lab-build.ps1
docker tag data_boar:lab fabioleitao/data_boar:1.7.1-beta
docker push fabioleitao/data_boar:1.7.1-beta
```

Prefer keeping **`fabioleitao/data_boar:latest`** aligned to **last stable** (**1.7.0**) until **1.7.1** final — see [DOCKER_IMAGE_RELEASE_ORDER.md](../ops/DOCKER_IMAGE_RELEASE_ORDER.md).
